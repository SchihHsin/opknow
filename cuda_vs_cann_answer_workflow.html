原因
两边问题其实都版本敏感（CUDA 的 PyTorch 扩展 API 跨版本变，CANN 随版本/芯片变），都不能凭记忆乱答；差的不是问题本身，而是我的先验强度 —— CUDA 资料在训练数据里极多，CANN 少。
怎么搜取决于二手源可信度：CUDA 可靠、CANN 稀缺，只能把可信度压到官方。
社区内容沉淀量差异 —— CUDA 十多年积累，优质内容铺满首屏；CANN 生态新，少。
CANN 官方文档排不到首屏，得显式补「官方」关键词，多花一轮；CUDA 第一屏就有。
搜索片段信息密度不同：CUDA 片段已能确证；CANN 摘要不足，才去尝试读全文。
工程链路长度不同 —— CUDA 靠 pybind11/cpp_extension 一条龙；CANN 多出 msOpGen / .run 打包 / OPP 部署。
不确定性累积 —— CANN 版本没锁定 + 官方全文没读全，折扣叠加，自评只能给「中」。
